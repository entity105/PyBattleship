# ShipWar

Цель работы заключалась в создании объектно-ориентируемой с графическим интерфейсом программы, реализующей известную игру «Морской бой». В качестве графического интерфейса была использована библиотека Tkinter. 

Описание работы программы
После запуска программы открывается окно «Меню», где есть кнопки «Выход», «Играть». По нажатии «Выход», программа завершается; по нажатии «Играть» открывается окно, где нужно расставить корабли по полю. После расстановки, для начала игры, нужно нажать на кнопку «В бой!». После чего открывается следующее окно – непосредственно с игровым процессом. Игра происходит между игроком (поле слева) и «компьютером» (алгоритмом). По окончании игры, (когда количество «потопленных» кораблей хотя бы на одном из двух 
полей == 10) отключается возможность кликать («стрелять»), и поверх текущего окна всплывает окно с соответствующем содержанием о победе или поражении.

Самыми непростыми задачами были:
-	Алгоритм (итеративный) автоматической расстановки кораблей (модуль «gamepole.py»). Полная расстановка происходит за 0.001397 секунд = 1.4 миллисекунды
-	Алгоритм умной стрельбы системы (модуль «gameplay.py»)
-	Интеграция с интерфейсом
Конечно, данный проект можно значительно расширить, добавив много различных фич и улучшений, таких как (планировалось): разные уровни сложности бота (разные алгоритмы стрельбы), регулируемые размеры поля (20x20, 50x50), разное количество кораблей (по умолчанию 10), радар, анимации и многое другое.
